Slack message event shape:
```typescript
interface SlackMessageEvent {
  type: "message";
  channel: string;        // DM channel ID
  user: string;           // User ID who sent it
  text: string;           // Message text (may be undefined for non-text msgs)
  ts: string;             // Message timestamp (unique ID)
  thread_ts?: string;     // Parent thread timestamp (undefined for top-level msgs)
  subtype?: string;       // "bot_message", "message_changed", etc. (undefined for normal user msgs)
  channel_type: "im";     // Always "im" for DM messages
}
```

  <behavior>
    ## Install dependency

    Run `cd infra/scheduler && npm install @slack/bolt` before writing code.
    Add `@slack/bolt` to package.json dependencies (npm install does this).

    ## Types

    - SlackMessage: { channel: string, user: string, text: string, ts: string, threadTs: string } (normalized — threadTs is always resolved)
    - SlackBotOptions: { botToken: string, appToken: string, onMessage: (msg: SlackMessage, reply: ReplyFn) => Promise<void>, onReaction?: (event: ReactionEvent) => Promise<void>, logLevel?: LogLevel }
    - ReplyFn: (text: string) => Promise<void> — sends a threaded reply
    - ReactionEvent: { reaction: string, user: string, itemChannel: string, itemTs: string }

    ## deriveConvKey(message: { channel: string, thread_ts?: string, ts: string }): string

    - Pure function, exported separately for testability
    - Returns `${message.channel}:${message.thread_ts ?? message.ts}`
    - This is the canonical thread identity key

    ## SlackBot class

    ### constructor(opts: SlackBotOptions)

    - Creates Bolt App with: { token: opts.botToken, appToken: opts.appToken, socketMode: true, logLevel: opts.logLevel ?? LogLevel.WARN }
    - Registers message handler via app.message():
      - Guard: if message.subtype exists OR message has no "text" field, return early
      - Guard: skip if message.channel_type is not "im" (safety: DMs only)
      - Resolve threadTs: message.thread_ts ?? message.ts
      - Build SlackMessage: { channel, user, text, ts, threadTs }
      - Build replyFn that calls say({ text, thread_ts: threadTs })
      - Call opts.onMessage(slackMessage, replyFn)
    - Registers reaction handler via app.event("reaction_added"):
      - If opts.onReaction is provided, call it with normalized event
    - Registers error handler via app.error():
      - Console.error with [slack-bot] prefix
      - Does NOT rethrow — keeps the bot running

    ### start(): Promise<void>

    - Calls await this.app.start()
    - Logs "[slack-bot] Connected via Socket Mode"

    ### stop(): Promise<void>

    - Calls await this.app.stop()
    - Logs "[slack-bot] Disconnected"

    ### getApp(): App

    - Returns the internal Bolt App instance (for testing or advanced configuration)
  </behavior>
