<feature>
  <name>WorktreeManager</name>
  <files>infra/scheduler/src/worktree.ts, infra/scheduler/src/worktree.test.ts</files>
  <behavior>
    ## Types

    - WorktreeConfig: { repoDir: string, maxWorktrees: number (default 4), worktreeDir?: string }
    - WorktreeInfo: { taskId: string, branch: string, path: string, allocatedAt: number, sessionId?: string }
    - WorktreeAllocResult: { ok: true, info: WorktreeInfo } | { ok: false, reason: "at-capacity" | "already-exists" | "git-error", error?: string }
    - WorktreeReleaseResult: { ok: true, merged: boolean, fallbackBranch?: string } | { ok: false, reason: "not-found" | "cleanup-error", error?: string }

    ## allocate(taskId, sessionId?)

    - Creates branch `worker/{taskId}` and worktree at `.worktrees/{taskId}` based on main
    - If branch `worker/{taskId}` already exists without a worktree, deletes it first (stale branch cleanup)
    - If a worktree already exists for this taskId, returns { ok: false, reason: "already-exists" }
    - If current allocations >= maxWorktrees, returns { ok: false, reason: "at-capacity" }
    - On git error, returns { ok: false, reason: "git-error", error: message }
    - Stores allocation in internal Map
    - Concurrent calls are serialized via Promise chain (no mutex library needed)

    ## release(taskId)

    - If taskId not in allocations Map, returns { ok: false, reason: "not-found" }
    - Auto-commits any uncommitted changes in the worktree (call autoCommitOrphanedFiles or equivalent)
    - Checks if worker branch has commits ahead of main (`git rev-list --count main..worker/{taskId}`)
    - If no new commits: removes worktree, deletes branch, returns { ok: true, merged: false }
    - If new commits:
      - Fetches origin/main
      - Rebases worker/{taskId} onto origin/main
      - On rebase success: checkout main, merge --ff-only worker/{taskId}, returns { ok: true, merged: true }
      - On rebase conflict: abort rebase, push branch as `session-{taskId}` fallback, returns { ok: true, merged: false, fallbackBranch: "session-{taskId}" }
    - Always cleans up: `git worktree remove --force`, `git branch -D`, removes from Map
    - Caller is responsible for pushing main via push queue after a successful merge

    ## list()

    - Returns array of current WorktreeInfo from internal Map
    - Synchronous (reads from memory)

    ## recover()

    - Runs `git worktree list --porcelain` and parses output
    - For each worktree with branch matching `worker/*` in `.worktrees/`:
      - Auto-commits orphaned files
      - Removes worktree with --force
      - Deletes branch
    - Runs `git worktree prune` as safety net
    - Returns count of recovered worktrees
    - Should be called on startup

    ## getCapacity()

    - Returns { current: number, max: number } from internal state

    ## Porcelain parser

    - parseWorktreeList(output: string): parses `git worktree list --porcelain` format
    - Each entry has: path, HEAD hash, branch (or null for detached HEAD)
    - Entries separated by blank lines

    ## Path resolution

    - Always resolve .worktrees/ as absolute path using repoDir
    - Use path.join(repoDir, ".worktrees", taskId) for worktree paths
  </behavior>
